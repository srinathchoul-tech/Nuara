# Custom Instructions & Rules

## Code Generation & Style
- **No Comments**: Do not include comments, docstrings, or explanatory notes inside any code files (e.g., no `//`, `#`, `/* ... */`, or `"""..."""`).
- **Clean & Human-like Code**: Write clean, natural, human-like code without verbose headers or AI-typical boilerplate to avoid AI detection.

## Git Operations
- **Commit Messages**: All git commit messages must strictly follow the sequential pattern: `initial commit 1`, `initial commit 2`, `initial commit 3`, etc.
